O que é NaiveBayes¶

link: https://www.vooo.pro/insights/6-passos-faceis-para-aprender-o-algoritmo-naive-bayes-com-o-codigo-em-python/

O que o algoritmo Naive Bayes? uma tcnica de classificao baseado no teorema de Bayes com uma suposio de independncia entre os preditores. Em termos simples, um classificador Naive Bayes assume que a presena de uma caracterstica particular em uma classe no est relacionada com a presena de qualquer outro recurso. Por exemplo, um fruto pode ser considerado como uma ma se vermelho, redondo, e tiver cerca de 3 polegadas de dimetro. Mesmo que esses recursos dependam uns dos outros ou da existncia de outras caractersticas, todas estas propriedades contribuem de forma independente para a probabilidade de que este fruto uma ma e por isso que conhecido como Naive (ingnuo).

O modelo Naive Bayes fcil de construir e particularmente til para grandes volumes de dados. Alm de simples, Naive Bayes conhecido por ganhar de mtodos de classificao altamente sofisticados.

Teorema de Bayes fornece uma forma de calcular a probabilidade posterior P (C | X) a partir de P (C), P (x) e P (X | c). Veja a equao abaixo:

probabilidade Bayes

Acima,

P (c | x) a probabilidade posterior da classe (c, alvo) dada preditor (x, atributos). P (c) a probabilidade original da classe. P (x | c) a probabilidade que representa a probabilidade de preditor dada a classe. P (x) a probabilidade original do preditor. Como o algoritmo Naive Bayes funciona? Vamos entender isso usando um exemplo. Abaixo eu tenho um conjunto de dados de treinamento de clima e da correspondente varivel-alvo Play (sugerindo possibilidades de jogar). Agora, precisamos classificar se os jogadores vo jogar ou no com base na condio meteorolgica. Vamos seguir os passos abaixo para realizar a operao.

Passo 1: Converter o conjunto de dados em uma tabela de frequncia

Passo 2: Criar tabela de Probabilidade ao encontrar as probabilidades de tempo Nublado = 0,29 e probabilidade de jogar = 0,64.

tabela bayes Passo 3: Agora, use a equao Bayesiana Naive para calcular a probabilidade posterior para cada classe. A classe com maior probabilidade posterior o resultado da previso.

Problema: Os jogadores iro jogar se o tempo est ensolarado. Esta afirmao est correta?

Podemos resolver isso usando o mtodo discutido acima de probabilidade posterior.

P (Sim |Ensolarado) = P (Ensolarado | Sim) * P (Sim) / P (Ensolarado)

Aqui temos P (Ensolarado | Sim) = 3/9 = 0,33, P (Ensolarado) = 5/14 = 0,36, P (Sim) = 9/14 = 0,64

Agora, P (Sim | Ensolarado) = 0,33 * 0,64 / 0,36 = 0,60, que tem maior probabilidade.

Naive Bayes usa um mtodo similar para prever a probabilidade de classe diferente com base em vrios atributos. Este algoritmo usado principalmente em classificao de texto e com os problemas que tm mltiplas classes.

Aplicações¶

Previses em tempo real: Naive Bayes um classificador de aprendizagem voraz e com certeza rpido. Assim, pode ser usado para fazer previses em tempo real.

Previses multi-classes: Este algoritmo tambm conhecido pela funcionalidade de previso multi-classes. Aqui podemos prever a probabilidade de mltiplas classes das variveis-alvo.

Classificao de textos/Filtragem de spam/Anlise de sentimento: classificadores Naive Bayes utilizados principalmente em classificao de textos (devido a um melhor resultado em problemas de classes mltiplas e regra de independncia) tm maior taxa de sucesso em comparao com outros algoritmos. Como resultado, amplamente utilizado na filtragem de spam (identificar spam) e Anlise de Sentimento (em anlise de mdia social, para identificar sentimentos positivos e negativos dos clientes)

Sistema de Recomendao: o classificador e a filtragem colaborativa Naive Bayes em conjunto constroem um sistema de recomendao que utiliza tcnicas de machine learning e minerao de dados para filtrar a informao invisvel e prever se um usurio gostaria de um determinado recurso ou no

Prós-Contras¶

Quais so os prs e contras de Naive Bayes? Prs:

  • fcil e rpido para prever o conjunto de dados da classe de teste. Tambm tem um bom desempenho na previso de classes mltiplas.
  • Quando a suposio de independncia prevalece, um classificador Naive Bayes tem melhor desempenho em comparao com outros modelos como regresso logstica, e voc precisa de menos dados de treinamento.
  • O desempenho bom em caso de variveis categricas de entrada comparada com a variveis numricas. Para variveis numricas, assume-se a distribuio normal (curva de sino, que uma suposio forte).

Contras:

  • Se a varivel categrica tem uma categoria (no conjunto de dados de teste) que no foi observada no conjunto de dados de treinamento, ento o modelo ir atribuir uma probabilidade de 0 (zero) e no ser capaz de fazer uma previso. Isso muitas vezes conhecido como Zero Frequency. Para resolver isso, podemos usar a tcnica de alisamento. Uma das tcnicas mais simples de alisamento a chamada estimativa de Laplace.
  • Por outro lado naive Bayes tambm conhecido como um mau estimador, por isso, as probabilidades calculadas no devem ser levadas muito a srio.
  • Outra limitao do Naive Bayes a suposio de preditores independentes. Na vida real, quase impossvel que ter um conjunto de indicadores que sejam completamente independentes.

DESVANTAGEM

  • NaiveByes trata cada coluna como independente o que nem sempre verdade

Perguntas do PDF ENlgish ML-AZ¶

Is Naive Bayes a linear model or a non linear model?

Naive Bayes is a non linear model. You will see that very clearly in Python or R when plotting the prediction boundary which will be a very nice curve well separating the non linearly distributed observations.

Can P (x) be zero?

Yes P (X) can be zero. That can happen when the new data point is an outlier, and therefore there is no other data points within the radius of the circle. However the formula in the Naive Bayes theorem is only true when P (X) is different than zero. When P (X) is equal to zero, then P (W alks|X) is just set to zero.

How does the algorithm decide the circle?

In the Intuition lecture we see that a circle is drawn to create a collection of data points similar to the new datapoint. The new datapoint was roughly at the center of the circle and hence we saw that number of green points were lesser than the number of red points and hence the new point went to the red category. But if we had drawn the circle a little differently around the new datapoint, then the number of green points could have been more than red. So how is that circle chosen? There is a parameter in the model that decides the radius of the circle, just like there is a parameter that chooses the number of neighbors in K-NN.

Naive Bayes vs K-NN?

The best way to answer that question is to try them both. Thanks to the templates it will only take a few minutes to compare their results. Just understand that the main difference between the two is that Naive Bayes is based on a probabilistic approach, unlike classic K-NN.

NaivieBayes in Python

What does the fit method do here?

It will simply train the Naive Bayes model on X_train and y_train. More precisely, the fit method will collect the data in X_train and y_train, and from that it will do all the successive operations you saw in the Intuition Lecture: first it will get the prior probability, the marginal likelihood, the likelihood and the posterior probability, and then using the circles around the observations it will learn how to classify these observations and predict future outcomes.

How to get the probabilities as output?

Here is the way to do that: y_proba = classifier.predict_proba(X_test) This will give you an array with 2 columns, one for 0 and one for 1.
